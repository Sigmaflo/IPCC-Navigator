## 동기

IEP-1001에서 단순 청킹 실험 결과 CASE 3(1000/200)과 CASE 4(1200/240)가 각각 0.8520, 0.8567로 최고 성능을 기록하였으나, 두 가지 한계가 확인되었다.

첫째, **사실 확인 유형**의 Context Recall이 CASE 3 기준 0.8627로 개선되었음에도, 고정 길이 방식의 구조적 한계(문맥 단절)가 근본적으로 해소되지 않았다. 둘째, CASE 4(1200)에서 NaN이 15개로 급증하여 청크 크기를 단순히 키우는 방식의 한계가 드러났다.

IPCC 보고서는 `섹션 → 숫자점숫자(X.X) → 숫자점숫자점숫자(X.X.X)` 3단계 계층 구조로 명확하게 조직되어 있다. 이 구조를 청킹 경계로 활용하면 임의적인 문맥 단절을 줄이고, 특히 **비교·의견/예측 유형**에서 성능 향상이 기대된다.

#### Step 3: 구조 패턴 분석 결과

**발견된 헤딩 패턴 및 최종 확정 수**

| 패턴명 | 정규식 | 최종 확정 수 | 비고 |
| :--- | :--- | :---: | :--- |
| 섹션번호 | `^섹션\s*\d+` | 46개 | 러닝헤더 포함 — Day 2에서 필터 필요 |
| SPM섹션 | `^SPM[.\s][A-Z]` | 0개 | 참조 줄 오탐 57개 전량 제거 |
| 숫자점숫자 | `^\d+\.\d+\s+[가-힣A-Z]` | 32개 | 핵심 절 단위 헤딩 |
| 숫자점숫자점숫자 | `^\d+\.\d+\.\d+\s+\S` | 28개 | 핵심 항 단위 헤딩 |
| CrossSection | `^Cross[-\s]Section` | 8개 | IPCC 특유의 통합 박스 |
| 부속서 | `^부속서\s*[IVX\d]+` | 23개 | p.135~178 구간 집중 |
| Box | `^Box\s*[\d.]+` | 3개 | 오탐 3개 제거 후 |
| 숫자점 | `^\d+\.\s+[가-힣A-Z]` | 2개 | |
| 알파벳점 | `^[A-Z]\.\s+[가-힣]` | 8개 | |
| **합계** | | **150개** | 오탐 57개 제거 (207→150) |

**오탐 제거 필터 목록**

```python
# 적용된 필터 (is_valid_heading_v2 기준)
1. 길이 3자 이하 → 제거 (페이지 번호, 단독 기호)
2. 순수 숫자 → 제거 (페이지 번호)
3. 120자 초과 → 제거 (2단 레이아웃 합침 줄)
4. {WGI .../WGII ...} 패턴 포함 → 제거 (본문 내 참조)
5. ^(그림|표)\s*[\d.]+ → 제거 (그림·표 캡션)
6. ^[A-Z]{2,},\s+[A-Z][a-z] → 제거 (판권 페이지 저자 목록)
7. SPM섹션 + WGII/WGIII/SR1.5 포함 → 제거 (참조 줄 오탐)
8. Box + 쉼표 직후 패턴(Box.N,) → 제거 (본문 내 Box 참조)
```

**저장 파일**: `ipcc_headings.json` (150개 헤딩, Day 2 청킹 입력값)

#### Day 1에서 발견한 Day 2 주의사항

- **목차 페이지(p.13~14) 헤딩 중복**: 동일 헤딩이 목차와 본문에 각각 등장 → 목차 페이지 skip 처리 필요
- **섹션번호 러닝헤더**: `섹션 2`, `섹션 3`처럼 제목 없는 섹션번호는 매 페이지 반복되는 러닝헤더 → 청크 구분자에서 제외 필요. `섹션 2: 현황 및 추세`처럼 콜론+제목이 붙은 것만 실제 경계로 사용
- **잔여 오탐 3개 수동 제거 필요**: `p.33 그림 캡션`, `p.99 그림 범례`, `p.100 본문 참조`
