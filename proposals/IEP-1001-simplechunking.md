## 분석

### 청크 크기와 Recall의 관계

청크 크기가 커질수록 Context Recall이 전반적으로 향상되며, **1000~1200 구간에서 최적점**이 존재한다. 1200을 초과하면 Recall이 다시 하락하고 NaN이 급증하는 역전 현상이 나타난다. 특히 사실 확인 유형은 CASE 3에서 Baseline 대비 +0.27로 가장 큰 폭의 개선을 보였다.

CASE 4(1200)가 Recall 0.8567로 수치상 최고이나, NaN 15개로 신뢰도가 낮다. NaN 안정성(6개)을 고려하면 **CASE 3이 최적**이다.

### 4지표 전체 평가 결과 (CASE 3 기준)

- **Context Precision 0.6117**: IEP-1002(0.2150) 대비 +0.3967, IEP-1003(0.5544) 대비 +0.0573으로 3종 중 가장 높음
- **Precision NaN 0개**: 100개 전 샘플 유효 — 신뢰도 완전 확보
- **의견/예측 Recall 0.9757**: 전 실험 통틀어 최고 수치
- **사실 확인 Precision 0.9133**: 사실 확인 유형의 검색 정밀도 탁월
- **범위 밖 Precision 0.0800**: 취약 구간 — IEP-2001 하이브리드 검색으로 개선 예정
- Faithfulness/Answer Relevancy는 llama3.1:8b 기준이므로, 배포 후 Claude haiku로 재측정 필요
