## 동기

IEP-2001 하이브리드 검색 실험에서 Context Precision이 0.6117 → 0.6406으로 소폭 개선됐으나 거절 로직 이슈(17/100)로 Recall이 급락했다. 이 과정에서 **"벡터 단독 검색이 이미 Recall 0.85로 높은 상황에서 BM25 추가가 실질적 이득을 주는가"** 에 대한 의문이 생겼다.

리랭킹은 BM25와 다른 접근이다. 검색 결과를 새로 추가하는 게 아니라 **이미 검색된 청크들을 질문과의 관련성 기준으로 재정렬**한다. 거절 로직을 건드리지 않고 Precision만 개선할 수 있다는 가설을 검증한다.

## 실험 설계

**파이프라인**

```
[IEP-1001 벡터 단독]
질문 → 벡터 검색 TOP_K(3) → Solar 답변

[IEP-2002 벡터 + 리랭킹]
질문 → 벡터 검색 CANDIDATE_K(10) → FlashRank 재정렬 → TOP_K(3) → Solar 답변
```

**거절 로직**: IEP-1001과 완전히 동일하게 TOP_K(3) 기준으로 threshold 판단. IEP-2001의 CANDIDATE_K 기준 방식 사용 안 함.

**실험 도구**: FlashRank (로컬 실행, API 불필요)

## 진행

### RAGAS 측정 전 모델 검증 (Cell 6)

RAGAS 전에 리랭킹 모델이 한국어 문서에서 실제로 의미 있는 순위 변화를 만드는지 먼저 검증했다. 유형별 질문 5개(사실확인 2, 비교 1, 의견예측 1, 경계케이스 1)로 두 가지 실험을 진행했다.

**실험 1 — 순위 변화 자동 집계**

두 모델 모두 5/5에서 순위 변경, 4/5에서 새 청크 등장이 확인됐다. 수치상으로는 모델이 변별력 있어 보였다.

**실험 2 — 사람이 직접 판단**

| 질문 | 유형 | MiniLM | MultiBERT |
| :--- | :---: | :---: | :---: |
| 2011~2020년 온도 상승 | 사실확인 | ❌ 악화 | ❌ 악화 |
| 1.5°C 탄소 감축량 | 사실확인 | 🟡 유사 | 🟡 유사 |
| 해수면 상승 비교 | 비교 | ❌ 악화 | 🟡 유사 |
| 기후변화 생태계 영향 | 의견예측 | ❌ 악화 | ❌ 악화 |
| 환경 보호 개인 실천 | 경계케이스 | ✅ 거절 유지 | ✅ 거절 유지 |

**두 모델 모두 "기후변화 생태계 영향" 질문에서 page 136(용어집)을 1위로 올렸다.** 벡터 단독 1위였던 page 61(생태계 영향 직접 서술)이 순위에서 사라졌다.

## 분석

### 핵심 문제 — 용어집·부속서 청크의 오판

두 모델 교체로도 해결되지 않은 문제가 있다. page 136은 용어집이다. "생태계", "영향", "기후변화" 같은 단어가 **정의 형태로 밀집**되어 있어 리랭킹 모델이 키워드 밀도를 높은 관련성으로 오판한다.

IPCC 보고서 구조 특성상 용어집(부속서 I), 약어집(부속서 II)에 전문용어가 밀집되어 있다. 실제 내용 청크와 메타 청크(용어집, 부속서)가 같은 색인에 혼재하는 상황에서 리랭킹 모델은 이를 구분하지 못한다.

### 모델 교체의 한계

| 항목 | MiniLM | MultiBERT |
| :--- | :--- | :--- |
| 학습 데이터 | 영어 MS MARCO | 다국어 MS MARCO |
| 한국어 처리 | 영어 임베딩으로 한국어 처리 | 다국어 지원 |
| 용어집 오판 | 동일하게 발생 | 동일하게 발생 |
| 전반적 품질 | 3/5 악화 | 2/5 악화 |

MultiBERT가 MiniLM보다 소폭 나은 케이스가 있었지만, 핵심 문제인 용어집 오판은 해결되지 않았다. 이는 **모델 성능 문제가 아니라 문서 구조 문제**다.

### 벡터 검색이 이 문제를 덜 겪는 이유

벡터 검색(jhgan/ko-sroberta-multitask)은 의미 기반 유사도를 측정하므로 키워드 밀도에 덜 민감하다. "생태계 영향"이라는 질문에 대해 용어 정의보다 실제 내용 서술을 더 관련 있게 평가한다. IEP-1001 Recall 0.8537이 이미 높은 이유가 여기 있다.

### 리랭킹이 효과적인 조건 vs 지금 상황

| 항목 | 리랭킹 효과적인 조건 | IPCC 프로젝트 현재 상황 |
| :--- | :--- | :--- |
| 벡터 Recall | 낮아서 개선 필요 | 이미 0.85로 높음 |
| 문서 구성 | 균일한 본문 청크 | 본문 + 용어집 + 부속서 혼재 |
| 언어 | 영어 또는 잘 지원되는 언어 | 한국어 (MS MARCO 학습 데이터 미포함) |
| 청크 크기 | 작아서 키워드 집중 | 1000자로 커서 키워드 밀도 균일 |

## 결론

**리랭킹 방식은 현재 IPCC RAG 파이프라인에서 효과 없음으로 결론.**

모델 교체(MiniLM → MultiBERT)로도 핵심 문제가 해결되지 않았으며, RAGAS 전체 측정 진행 없이 Cell 6 검증 단계에서 중단한다. 이유는 세 가지다.

첫째, 사람이 직접 판단했을 때 벡터 단독이 대부분의 경우 더 좋거나 동등하다. 둘째, 용어집·부속서 오판 문제는 리랭킹 모델 교체로 해결되지 않으며, 해결하려면 색인 단계에서 메타 청크를 제외하는 전처리가 필요하다. 셋째, COCOON 마감(5/15)을 고려할 때 이 문제를 해결하는 것보다 배포(IEP-4001~4003)에 집중하는 것이 우선이다.
