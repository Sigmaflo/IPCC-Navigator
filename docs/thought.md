## 📚 [문헌 연구] JCCR 논문 분석 및 전략 고도화

김선회·윤순진(2025), "IPCC 정책결정자를 위한 요약 보고서의 계량 텍스트 분석" 논문을 통해 도출한 기술적/기획적 인사이트 정리.

### 1. ⚙️ 데이터 엔지니어링: 청킹(Chunking) 전략
- **문제점:** SYR SPM은 텍스트가 매우 압축적(약 378문장)이라 검색 효율이 낮을 수 있음.
- **해결책:** 논문의 분석 단위를 참고하여 **'5개 문장 단위 묶음(5-sentence Chunk)'**을 기본 청킹 단위로 설정.
- **기대효과:** 문맥(Context)을 보존하면서도 통계적 유의성을 확보한 검색 결과 도출 가능.

### 2. 🎭 페르소나 및 프롬프트: IPCC 6차 보고서의 톤앤매너
- **메시지 중심 이동:** '물리적 현상 증명' → **'정책적 실천 및 행동 촉구'**로 답변 톤 조정.
- **용어의 직관성:** - 학술적 용어(`anthropogenic`)보다 직관적이고 책임 소재가 명확한 단어(`human activity`, `human-caused`) 우선 사용 유도.
  - 답변 생성 시 "인류의 활동이 기후 변화의 주된 원인임"을 명료하게 전달하도록 프롬프트 설계.

### 3. 🔑 검색 최적화를 위한 키워드 사전 (Metadata/Keyword)
- **가중치 부여 키워드 (Hot Keywords):**
  - 목표: `1.5°C`, `2°C`, `탄소중립(net-zero)`, `한계초과(overshoot)`
  - 정책: `기후회복적 발전`, `재원(finance)`, `포용적(inclusive)`, `공동편익(co-benefit)`, `SDG`, `NDC`
- **지양/퇴조 키워드 (Cold Keywords):**
  - `SRES`, `안정화(stabilization)`, `Annex B`, `인위적인(anthropogenic)`
- **반영 계획:** RAG 검색 시 위 'Hot Keywords'가 포함된 문서에 가중치를 주는 가중 검색(Weighted Search) 검토.

## 📅 2026-02-24: GraphRAG 및 TDA 결합 전략 및 비용 최적화 로드맵

## 1. 프로젝트 개요
- **목표**: IPCC AR6 보고서와 같은 방대한 비정형 데이터를 GraphRAG와 TDA(위상 데이터 분석)를 결합하여 분석하고, 이를 웹 서비스로 구현한다.
- **핵심 차별점**: 단순한 텍스트 검색을 넘어 데이터 간의 명시적 관계(GraphRAG)와 고차원적 형상(TDA)을 동시에 시각화하고 분석에 활용한다.

## 2. 현재까지의 기술적 진전
- **GraphRAG 기초 구현**: `NetworkX`를 활용해 엔티티 간의 관계를 정의하고 탐색하는 기본 로직 확인 완료.
- **TDA 시각화 성공**: `KeplerMapper`를 사용하여 고차원 데이터(Digits)를 위상망 형태로 시각화 성공.
  - *Troubleshooting*: 데이터 스케일링(MinMaxScaler)과 클러스터링 파라미터(eps, n_cubes) 조정의 중요성 확인.
- **환경 구성**: VS Code 내 가상환경에서 Jupyter Notebook을 활용한 개발 환경 구축 완료.

## 3. 주요 전략적 결정
### 비용 최적화 (Zero-Cost 전략)
- **MS GraphRAG 대안**: MS GraphRAG의 높은 인덱싱 비용(LLM API 호출)을 피하기 위해 **커스텀 파이프라인** 구축을 우선 고려한다.
- **LLM 활용**: 고비용 모델 대신 **Gemini 1.5 Flash 무료 티어**를 관계 추출 및 최종 답변 생성에 활용한다.
- **로컬 임베딩**: 벡터 변환은 `HuggingFace`의 오픈소스 모델을 로컬(CPU)에서 실행하여 비용을 완전히 제거한다.

### 하드웨어 대응
- **로컬 LLM 지양**: GPU 사양의 한계로 인해 Ollama 등의 로컬 LLM 구동보다는 API 기반(Gemini) 처리를 선택한다.
- **경량 파이프라인**: 데이터 전체를 한 번에 처리하기보다 샘플 데이터를 활용한 단계적 인덱싱을 수행한다.
